You are a senior backend architect. Extend the existing Fastify-based Node.js backend to implement **multi-language AST-based chunking** for:

* JavaScript / TypeScript
* Java
* C# (.NET)
* SQL

This must be implemented using a **pluggable architecture**, not a single parser.

---

# 🎯 OBJECTIVE

Enable the system to:

* Parse multiple languages
* Extract meaningful code units (functions, classes, APIs, SQL objects)
* Normalize them into a common chunk format
* Store them in ChromaDB

---

# 🧠 CORE DESIGN

## 1. Main Orchestrator

astChunker.service.ts

Function:

extractChunks(filePath, content)

* Detect language from extension:

  * .js, .ts → JS chunker
  * .java → Java chunker
  * .cs → C# chunker
  * .sql → SQL chunker

* Call respective chunker

* Return normalized chunks

---

# 📦 COMMON CHUNK FORMAT

```ts
{
  id: string,
  content: string,
  metadata: {
    type: string,
    name: string,
    file: string,
    language: string,
    route?: string,
    method?: string
  }
}
```

---

# 🔧 LANGUAGE-SPECIFIC IMPLEMENTATION

## JS/TS (Babel-based)

Extract:

* FunctionDeclaration
* Arrow functions
* Class methods
* Express/Fastify routes

---

## Java (tree-sitter)

Extract:

* class_declaration
* method_declaration

Metadata:

* class name
* method name

---

## C# (.NET)

Extract:

* class_declaration
* method_declaration

Also detect:

* Controllers (class name ends with Controller)
* API routes (attributes like [HttpGet], [HttpPost])

---

## SQL

Extract:

* CREATE TABLE
* CREATE PROCEDURE
* CREATE FUNCTION

Metadata:

* object type
* name

---

# ✂️ CONTENT EXTRACTION

# 🔌 INTEGRATION

## Update ingestion pipeline

For each file:

```ts
const chunks = astChunker.extractChunks(filePath, content);
await chromaService.upsertDocuments(projectId, chunks);
```
