You are a senior software architect. Extend the existing Fastify-based Node.js system to implement **Auto High-Level Design (HLD) generation from API ↔ DB lineage using RAG (Chroma + LLM)**.

This feature must generate structured HLD documents based on system understanding derived from lineage and code context.

---

# 🎯 OBJECTIVE

Generate a High-Level Design (HLD) that includes:

* System overview
* Logical modules/services
* API grouping per module
* Database mapping
* Data flow between modules
* Architecture suggestions

---

# 🧱 ARCHITECTURE ADDITION

Create:

/modules/hld
hld.controller.ts
hld.service.ts

---

# 🧠 INPUT DATA SOURCES

Use:

* api_table_map (lineage data)
* API definitions
* DB schema
* Chroma (for semantic context)

---

# ⚙️ HLD GENERATION FLOW

## Step 1: Aggregate Data

For a project:

* Group APIs by:

  * table usage
  * naming similarity

* Build clusters:

  * APIs touching same tables → same module

---

## Step 2: Retrieve Context (Chroma)

Query:

* “authentication logic”
* “order processing”
* “user management”

Retrieve:

* relevant functions
* business logic

---

## Step 3: Build Prompt (MANDATORY STRUCTURE)

Use prompt.service with versioning

---

## PROMPT TEMPLATE

```text id="pn6v8o"
You are a senior system architect.

Given the following system data:

API to DB lineage:
{lineage}

Database schema:
{schema}

Code context:
{context}

Generate a High-Level Design (HLD) with:

1. System Overview
2. Modules/Services (group APIs logically)
3. API mapping per module
4. Database tables per module
5. Data flow between modules
6. Architecture recommendation

Return STRICT JSON:

{
  "overview": "string",
  "modules": [
    {
      "name": "string",
      "apis": ["string"],
      "tables": ["string"]
    }
  ],
  "dataFlow": [
    "Module A → Module B"
  ],
  "architecture": "string"
}
```

---

## Step 4: LLM CALL (MANDATORY)

* Retry up to 3 times
* Validate JSON strictly

---

## Step 5: POST-PROCESSING

* Normalize module names
* Remove duplicates
* Validate:

  * each API appears in a module
  * each table is mapped

---

## Step 6: STORE RESULT

Table:

documents

Fields:

* type: "HLD"
* content: JSON
* project_id
* version
* created_at

---

# 🔌 API ENDPOINTS
